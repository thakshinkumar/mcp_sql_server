# Ollama SQLCoder Integration Guide

## Overview
Successfully integrated Ollama SQLCoder 7B model for local, open-source NL-to-SQL conversion.

## What is Ollama SQLCoder?
- **Model**: SQLCoder 7B (4.1 GB)
- **Type**: Open-source, locally-hosted LLM specialized for SQL generation
- **Advantage**: No API costs, no rate limits, runs completely offline
- **Performance**: ~10-45 seconds per query generation

## Installation Steps

### 1. Install Ollama
Download and install from: https://ollama.com/download

### 2. Verify Installation
```bash
ollama --version
```

### 3. Pull SQLCoder Model
```bash
ollama pull sqlcoder:7b
```

### 4. Verify Model Downloaded
```bash
ollama list
```
You should see `sqlcoder:7b` in the list.

### 5. Test Ollama Server
```bash
curl http://localhost:11434/api/tags
```

## Configuration

### Update `.env` File
```env
# LLM Configuration
LLM_PROVIDER=ollama
LLM_API_KEY=
LLM_ENDPOINT=http://localhost:11434
LLM_MODEL=sqlcoder:7b
LLM_TEMPERATURE=0.3
LLM_MAX_TOKENS=500
```

## How It Works

### Architecture
1. **User Query** → Natural language input
2. **Ollama SQLCoder** → Generates primary SQL query (10-45 seconds)
3. **Variation Generator** → Creates 2 additional SQL variations instantly
4. **Cost Evaluator** → Evaluates all 3 candidates
5. **RL Agent** → Selects best query based on learned patterns
6. **Execution** → Runs selected query on SQL Server

### Optimization Strategy
- **Single Ollama Call**: Only 1 query generated by Ollama (slow but accurate)
- **Fast Variations**: 2 additional queries created by rule-based variations (instant)
- **Result**: 3 candidates total with minimal latency

### Prompt Template
SQLCoder uses a specialized prompt format:
```
### Instructions:
Convert this question into a SQL query for SQL Server.

### Input:
{user_question}

### Database Schema:
{schema_info}

### Response (SQL only):
```

## Troubleshooting

### Issue: Timeout Errors
```
HTTPConnectionPool(host='localhost', port=11434): Read timed out
```

**Solution**: Ollama is slow on first run. Wait 60 seconds or:
- Reduce `num_predict` in options
- Use smaller model: `ollama pull sqlcoder:3b`
- Increase timeout in code

### Issue: "SHOW COLUMNS" Syntax
```
Skipping non-SQL text: SHOW COLUMNS FROM CUSTOMERS;
```

**Solution**: SQLCoder sometimes generates MySQL syntax. The system automatically:
- Detects invalid syntax
- Falls back to mock mode
- Generates valid SQL Server queries

### Issue: Ollama Not Found
```
ollama: command not found
```

**Solution**: Add Ollama to PATH or use full path:
```bash
# Windows
$env:LOCALAPPDATA\Programs\Ollama\ollama.exe list

# Or add to PATH permanently
```

## Advantages of Ollama

1. **No API Costs**: Completely free, unlimited usage
2. **Privacy**: All data stays local, no external API calls
3. **Offline**: Works without internet connection
4. **No Rate Limits**: Generate unlimited queries
5. **SQL-Specialized**: Trained specifically for SQL generation
6. **Customizable**: Can fine-tune model for your schema

## Limitations

1. **Slow**: 10-45 seconds per query (vs 2-5s for cloud APIs)
2. **Resource Intensive**: Requires 8GB+ RAM
3. **Model Size**: 4.1 GB disk space
4. **Occasional Errors**: May generate wrong SQL dialect

## Future Improvements

1. **Model Caching**: Keep model in memory for faster responses
2. **Streaming**: Use streaming API for progressive results
3. **Fine-tuning**: Train on your specific database schema
4. **Smaller Model**: Use `sqlcoder:3b` for faster responses
5. **GPU Acceleration**: Use CUDA for 10x faster generation
